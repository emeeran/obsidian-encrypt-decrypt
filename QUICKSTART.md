# Quick Start Guide

## 🚀 Get Started in 2 Minutes

### 1. Installation
- **Community Plugins**: Search "Note Encryptor" in Obsidian Community Plugins
- **Manual**: Download from GitHub and extract to `.obsidian/plugins/note-encryptor/`

### 2. Encrypt Your First Note

#### Method 1: Ribbon Icon (Easiest)
1. Open any note
2. Click the 🔒 lock icon in the ribbon
3. Enter a strong password (min 8 characters)
4. Click **Encrypt**

#### Method 2: Command Palette
1. Press `Ctrl/Cmd + P`
2. Search "Encrypt current note"
3. Follow password prompts

### 3. Decrypt Your Note
1. Open the encrypted note (has 🔒 prefix)
2. Click the 🔒 lock icon or use command palette
3. Enter your password
4. Click **Decrypt**

## 🔧 Quick Settings

### Essential Settings
```
Settings → Community Plugins → Note Encryptor → Options

✓ Password Requirements: Enable strong passwords
✓ File Size Limit: 10MB (recommended)
✓ Visual Indicators: Show password strength
✓ Confirmation: Enable for sensitive notes
```

### Security Tips
- **Password Length**: 12+ characters recommended
- **Password Complexity**: Mix uppercase, lowercase, numbers
- **Password Management**: Use password managers
- **Backup Strategy**: Regular backups of encrypted notes

## 📁 File Naming

### Default Behavior
```
Original: "My Secret Note.md"
Encrypted: "🔒 My Secret Note.md"
Decrypted: "My Secret Note.md"
```

### Custom Prefix/Suffix
```
Settings → Note Encryptor → File Naming
Prefix: "[ENCRYPTED] " or "🔒 "
Suffix: " (LOCKED)" or "[SECURE]"
```

## 🚨 Troubleshooting

### Common Issues

| Problem | Solution |
|---------|----------|
| "Encryption failed" | Check file size, reduce content |
| "Wrong password" | Verify exact password (case-sensitive) |
| File naming conflict | Plugin automatically adds numbers |
| Note won't decrypt | Check for file corruption |

### Recovery Options
```
⚠️ Critical: Lost passwords CANNOT be recovered!

✅ Prevention:
- Test with non-critical data first
- Use password managers
- Keep secure password backups
```

## 🔒 Security Checklist

### Password Security
- [ ] At least 12 characters
- [ ] Contains uppercase letters
- [ ] Contains lowercase letters
- [ ] Contains numbers
- [ ] Different from other passwords
- [ ] Stored securely (password manager)

### Backup Strategy
- [ ] Regular encrypted note backups
- [ ] Test restore procedures
- [ ] Multiple backup locations
- [ ] Password backup plan

## 🎯 Best Practices

### Do's ✅
- Use unique passwords per note
- Test encryption with sample data
- Keep regular backups
- Update plugin regularly
- Use password managers

### Don'ts ❌
- Don't reuse passwords
- Don't store passwords in plain text
- Don't share passwords
- Don't ignore update notifications
- Don't modify encrypted files manually
